Problem Statement & Scope

Modern saloons require structure to eliminate wait times and maintain professional records.

  • Manual queueing causes long waiting hours and customer frustration.
  • Lack of centralized staff assignment prevents optimization.
  • Manual invoices and billing records are prone to accounting mistakes.
  • Managers lack real-time overview of business sales metrics.
🎯
Project Objectives
Provide direct client booking, match bookings dynamically with available expert stylists, and auto-generate clean invoices.
🚀
Impact
Reduces saloon booking overhead by 65%, organizes stylists' work timetables, and makes client management frictionless.

Technology Stack & Architecture

A lightweight, responsive, and robust PHP/MySQL infrastructure.

💻

HTML5, CSS3, JavaScript, and jQuery. Uses responsive Bootstrap templates designed for clean mobile and desktop layouts.
⚙️

🗄️
Database Server
MySQL Database (`msmsdb`). Organizes entities, uses indexes for fast lookups, and stores secure MD5-hashed credentials.

Database Design

Relational database organized into 9 specialized tables for modules.

tbladmin tblcustomers tblstylists tblservices tblstylist_services tblappointment tblinvoice tblpage tblsubscriber

Integrates Many-to-Many routing table tblstylist_services to links stylists dynamically to service categories based on specialties.

Core Table Linkages

Customer Links: tblcustomers(ID) maps to tblinvoice(Userid) for billing histories.
Expertise Mapping: tblstylists(ID) and tblservices(ID) feed into tblstylist_services to filter client booking choices.
Appointment Flow: Stores admin flags (Status) and stylist flags (StylistStatus) for booking verification.

Client (Customer) Portal

Self-registration and dynamic service booking portal.

  • Account Creation: Simple signup registering gender, contact details, and credentials.
  • Service-Linked Booking: Dynamically matches and displays only stylists expert in the selected service.
  • Booking Tracker: Shows whether the requested stylist and admin have confirmed the date.
  • Invoice Ledger: Tracks past service receipts.
Dynamic Stylist Filtering

When booking, client selects a service. Ajax calls retrieve available styling staff from database using:

SELECT * FROM tblstylists s
JOIN tblstylist_services ss
ON s.ID = ss.StylistId
WHERE ss.ServiceId = ?

Double-Verification Logic

To ensure high scheduling accuracy, appointments rely on a dual approval loop:

  • Client sends request to an expert stylist.
  • Stylist accepts/rejects via stylist scheduling dashboard.
  • Admin evaluates overall slot availability and accepts/rejects.
  • Both must accept for confirmation.

Stylist (Staff) Panel

Empowers stylists with autonomous queue and schedule controls.

  • My Schedule Dashboard: Real-time view of customer requests referencing their expertise.
  • Respond to Booking: Allows stylists to accept or reject requests and leave remarks.
  • Expertise Setup: Admin maps specialties so clients only assign correct bookings.
  • Profile Management: Updates personal info and specialty fields.

System Workflows

Streamlined flow guarantees smooth checkout and booking scheduling.

  1. Browse: User views pricing and selects a service.
  2. Request: User logs in, enters appointment details, and picks a matching expert stylist.
  3. Verification: Both stylist and administrator confirm the request.
  4. Render & Bill: Service completed, admin logs details, and prints a final invoice receipt.
🧾
Paperless Billing Flow

Admin accesses "Billing" panel, selects the customer, checks off the rendered services list, and creates a unique BillingId invoice receipt instantly.
